I've answered this question before but here we go again with some detail on why:Why Do We Need CoreXY OptionCOREXY is an option where the kinematics of the robot require that both motors turn at the same time to generate a straight line. If you were to build a non CoreXY robot where one motor controls X motion and another motor controls Y motion, you would have to mount one of the motors on the other axis which means it would move with the axis. The extra weight of the motor would slow the robot down. CoreXY is a way to route the belt so that both motors must move to achieve either an X or a Y motion. Both motors are stationary which greatly reduces the mass being moved.  If you fail to tell the software that you are using CoreXY with a CoreXY robot, you get a 45 degree motion when calling for either an X or Y motion. This is because only one motor is moving instead of the required 2 motors.Why is my Robot Still Plotting at 45 degreesThe reason that so many people struggle with getting CoreXY to work is that they are not editing the config.h file that the Arduino IDE is compiling. My instructions explain how to find the correct file location. It is not likely to be where you think it is. Turn on verbose mode while compiling and the Arduino IDE will log what file it is using. Edit that file to enable CoreXY.If your robot draws at a 45 degree angle, read this whole description before reporting this "problem". Please read this description.
